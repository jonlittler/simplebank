# CD/CI Continuous Integration/Deployment

### GitHub Actions

Create a **Workflow** \
Consists of 1+ or more jobs \
That can be triggered by events, schedule or manually

Requires a **Runner** \
Which is a server to run the jobs \
RUn 1 job at a time \
Reports status, logs and results

**Jobs** are a set of steps to execute \
That run in parallel \
Except if jobs depend on each other

**Steps** are individual tasks \
That run serially \
Consists of 1+ or more actions

**Action** is a standalone command \
That runs serially \
And can be reused
